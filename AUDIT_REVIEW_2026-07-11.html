🆕 复检新发现（7 项）

New Medium N-1. load_pdf_data / save_pdf_data 未做 PDF 路径授权与扩展名校验
文件：src-tauri/src/lib.rs:651-676
read_pdf_bytes 与 get_pdf_hash 入口均调用 validate_pdf_access（扩展名 .pdf + 授权白名单），但 load_pdf_data / save_pdf_data 仅经 annotations_path → compute_pdf_hash(file_path) 直接打开并读取任意路径文件以计算哈希，无任何校验。后果：在 webview 被 XSS 控制时，可对任意文件触发读取（用于哈希，存在文件存在性/内容相等性侧信道）并向可预测的标注文件写入 JSON。虽不泄露完整文件字节，但与既有安全模型不一致，属于遗漏的攻击面。
在 load_pdf_data / save_pdf_data 命令开头加 validate_pdf_access(state, &file_path)?，与 read_pdf_bytes / get_pdf_hash 保持同一安全基线；并补充 Rust 单测。
New Low N-2. shell:default 能力已授权但插件未被前端使用（多余攻击面）
文件：capabilities/default.json:10 + lib.rs:85 + Cargo.toml:27
初版 L-13 曾指出 tauri-plugin-shell 初始化却未授权；修订时反过来在 capabilities 中授予了 shell:default。但经核查，open_path / open_logs_dir 实际使用的是 Rust open crate，前端 JS 从未调用 @tauri-apps/plugin-shell。shell:default 会开放命令执行能力，属于不必要的攻击面。
若近期无 shell 调用计划，移除 shell:default 权限及 tauri-plugin-shell 依赖/插件初始化；如需保留供未来使用，应在报告中明确记录。
New Low N-3. 旧版明文 API Key 迁移时，若清空写盘失败会残留明文
文件：src-tauri/src/lib.rs:591-612
load_settings_with_storage 在迁移时先把 key 存入钥匙串，再调用 save_settings_to_disk(cleared) 清空磁盘字段。若本次清空写盘失败（磁盘满/权限），磁盘文件仍含明文 key，而钥匙串已有副本；后续加载从钥匙串命中、不再重清，明文永久残留。概率低，但属数据泄露边角。
先构造 cleared（清空 api_key）并成功写入磁盘后，再 storage.store；或在 store 前先以原子写覆盖为清空版本，确保“磁盘无明文”优先于“钥匙串有值”。
New Low N-4. Rust Annotation 结构体缺少 fileHash 字段
文件：src-tauri/src/lib.rs:268-291
前端 Annotation 含 fileHash（分屏过滤用），但 Rust 结构体无此字段。当前因标注按 PDF 文件（以内容哈希命名）分文件存储，fileHash 仅在内存用于过滤，序列化时被忽略、加载时回填，功能正确。但若未来将多 PDF 标注存入同一文件，fileHash 会静默丢失，埋下脆弱性。
在 Rust Annotation 增加 file_hash: String（带 rename_all 与 default），与前端保持一致，消除隐式耦合。
New Info N-5. CSP 仍允许 https: 通配（功能权衡，建议 LLM 代理化）
文件：tauri.conf.json:26
webview 直接持有 API Key 并向任意 https: LLM 端点发起 fetch（llm.ts）。一旦 webview 被 XSS，Key 可外发至任意 https 域名。当前因需支持自定义/自托管 LLM 端点而保留通配，是可接受的权衡。
中长期建议将 LLM 请求代理到 Rust 后端：webview 仅与本地后端通信，API Key 永不离后端，CSP 即可收紧为仅 https://localhost，从根本上消除 Key 外泄面。
New Low N-6. i18n 框架已接入，但 UI 文案仍仅中文
文件：package.json（i18next + react-i18next）、src/i18n、各组件使用 i18n.t。
框架与文案抽取已完成（满足 M-6 的工程要求），但实际仅提供中文本地化，尚无其他语言资源文件。对以中文为目标用户的产品可接受，但“国际化”能力尚未落地。
如需面向海外工程师，补齐 en / ja / de 等语言包；否则在文档中标注当前仅支持中文界面。
New Low N-7. 测试套件强壮，但核心 E2E 覆盖与 act() 告警可优化
文件：e2e/、App.test.tsx 等
前端 155 项单测 + Rust 序列化/安全单测 + CI 门禁均已就位。但初版指出的核心用户流程 E2E（打开 PDF → 选中 → 翻译/解读 → 持久化 → 重启恢复）在浏览器 E2E 中仍偏薄（当前 E2E 以布局/页码跳转为主）；且测试运行时出现 React act() 告警，建议收敛。
增补“重启后标注/会话恢复”的端到端 E2E（可借 Tauri 持久化目录注入）；为流式状态更新包裹 act() 消除告警。
