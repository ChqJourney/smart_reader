## 1. 产品概述

### 1.1 产品名称

**SpecReader AI**（暂定，内部代号：PDF Standard Agent）

### 1.2 一句话描述

一款面向检测认证工程师的桌面端 AI 助手，用于降低阅读、理解和执行各类标准 PDF 文件时的认知与智力开销。

### 1.3 产品定位

不是通用 PDF 阅读器，也不是通用翻译工具，而是**标准文件阅读理解副驾驶**。核心目标是把工程师从“反复查术语、跳条款、翻译长句、整理测试要求”的低效工作中解放出来，让他们更快、更准地把标准转化为可执行的测试方案。

### 1.4 核心价值主张

- **读懂**：专业术语自动解释，跨语言实时翻译，复杂长句结构化解读。
- **找到**：条款引用一键追踪，上下文快速定位。
- **用起来**：自动提取测试要求，生成可导出的检测清单。

## 2. 目标用户与场景

### 2.1 目标用户

**主要用户**：检测认证工程师（Testing & Certification Engineer）

典型画像：

- 工作涉及 IEC、ISO、EN、GB、UL、ASTM、IEEE 等标准文件。
- 需要阅读英文或其他外语标准，专业术语密集。
- 经常需要跨条款追踪引用关系（如 "see Clause 5.2.3"）。
- 最终产出为测试计划、测试报告、合规判定结论。
- 对数据隐私敏感，标准文件通常不希望上传到公共云。

### 2.2 典型使用场景

| 场景         | 用户行为                                | 产品价值                                                  |
| ------------ | --------------------------------------- | --------------------------------------------------------- |
| 初读标准     | 打开一份 200 页英文标准，不知道重点在哪 | AI 自动生成文档摘要、章节导航、术语表                     |
| 精读条款     | 读到一段复杂描述，不确定具体含义        | 选中后 AI 解读，说明“这条要求什么、为什么、怎么测”        |
| 跨条款追踪   | 看到 "refer to Clause 4.2" 需要跳转     | 点击引用直接跳转，并显示引用上下文                        |
| 术语理解     | 遇到缩写或专业词，不确定定义            | 悬停显示定义，自动关联 Terms and definitions 章节         |
| 表格信息查询 | 需要确认某个测试等级表的具体数值        | AI 提示表格位置，必要时截图调用多模态 LLM 精确读取        |
| 整理测试方案 | 需要把标准转化为测试项目清单            | AI 提取 Shall/Must/Requirements，生成 Excel/Markdown 清单 |
| 多语言标准   | 阅读德/葡/西/日标准                     | 按用户设置的源语言与目标语言翻译                          |

## 3. 产品目标

### 3.1 MVP 目标（超轻量版）

在本阶段（内部称「超轻量版」），产品聚焦工程师阅读标准 PDF 时最高频的两个痛点：**翻译**与**解读**。暂不做条款索引、引用追踪、术语表、测试清单等需要结构化文档数据的功能。

> **打开 PDF → 本地渲染 → 选中文本 → AI 翻译 / 解读**

### 3.2 关键成功指标（超轻量版）

| 指标                 | 目标   | 说明                                 |
| -------------------- | ------ | ------------------------------------ |
| 文档打开到可阅读时间 | < 5 秒 | PDF 渲染完成，可滚动、选区           |
| 选中到 AI 响应时间   | < 3 秒 | 流式输出首字符延迟                   |
| 翻译可读性           | > 85%  | 工程师无需二次猜测即可理解           |
| 解读可用率           | > 80%  | 解释能说明“要求什么、为什么、怎么测” |

## 4. 设计原则

1. **来源可追溯**：AI 回答尽量标注来源页码（由前端在选中文本时记录），支持跳回原文。
2. **本地优先**：PDF 文件在本地渲染、本地查看，不上传完整文件。
3. **用户控制 AI**：用户自备 OpenAI 兼容 API Key，自主选择模型和供应商。
4. **轻量运行**：本阶段不做 PDF 转 Markdown、不做 Clause 索引、不做结构化解析。
5. **专业翻译**：术语保留原文，提供行业解释，不做通用直译。
6. **少即是多**：超轻量版只聚焦翻译与解读，其他功能后续迭代。

### 5.3 超轻量版功能范围

本阶段（超轻量版）已实现以下功能，其他需求延后：

- **PDF 阅读器**：打开本地 PDF、文本型 PDF 渲染、单页/连续滚动阅读模式、页码跳转与缩放、文本选区、多 PDF Tab（最多同时打开 10 个文件）。
- **AI 实时辅助面板**：实时翻译（页面内浮层批注）、选中解读（右侧面板条目列表）、自定义解读（基于暂存区多片段）、会话追问、流式输出、Markdown 渲染、**悬停取词翻译（基于本地 ECDICT 词典，设置中可开关）**。
- **暂存区**：选中文本后可「加入暂存」，暂存片段汇聚在右侧面板；支持编辑、删除、清空、跳转回原文；可选取多个片段发起自定义解读。
- **批注持久化**：翻译浮层、解读标记、已解读暂存标记的位置和内容按 PDF 文件 hash 持久化到 AppData；解读会话以独立 JSON 文件保存，重新打开同一文件自动恢复。
- **设置与配置**：API 配置（Base URL、API Key、Model）、目标语言、悬停取词翻译开关。
- **布局交互**：左右分栏宽度可鼠标拖拽调节，左右面板均可隐藏/显示；选区工具条点击外部自动消失。

明确不在本阶段实现：

- PDF 文本提取与 Clause 索引生成。
- 目录/大纲导航、全文搜索。
- 术语表自动提取、引用追踪、测试清单生成（悬停取词翻译已以 ECDICT 离线词典形式实现，不等同于结构化术语表）。
- License 激活（可先保留配置入口，但不强制校验）。
- 表格精确读取（暂不实现多模态截图）。

## 6. 用户故事与用例

### 6.1 用户故事 1：快速理解一份新标准

> 作为一名认证工程师，当我首次打开一份 150 页的英文标准时，我希望在 1 分钟内看到文档结构、核心主题和关键术语，以便快速判断需要重点阅读哪些章节。

**验收标准**：

- 打开 PDF 后自动解析并显示大纲。
- 右侧面板显示文档级摘要。
- 自动提取术语表并在原文中高亮。

### 6.2 用户故事 2：理解复杂条款

> 作为一名认证工程师，当我读到一段包含多个条件从句的测试要求时，我希望选中后 AI 能告诉我这条到底在测什么、需要哪些设备、结果怎么判定。

**验收标准**：

- 选中文本后弹出浮动工具栏。
- 点击“解读”后 AI 输出结构化解释。
- 解释包含：要求概述、测试条件、判定依据、相关条款。

### 6.3 用户故事 3：追踪条款引用

> 作为一名认证工程师，当我看到 "The test shall be carried out in accordance with Clause 5.2.3" 时，我希望点击 "Clause 5.2.3" 就能跳转到对应位置，而不是手动翻页。

**验收标准**：

- 引用文本可点击或可识别为交互元素。
- 点击后 PDF 滚动到对应位置并高亮。
- 若引用目标未解析到，给出明确提示。

### 6.4 用户故事 4：查询表格精确信息

> 作为一名认证工程师，当我问 AI 某个测试等级的具体电压值时，我希望 AI 能准确读取表格内容并告诉我数值，而不是编造。

**验收标准**：

- AI 识别到问题涉及表格时，优先说明表格位置。
- 若用户需要精确值，AI 自动截取表格区域图片。
- 调用支持视觉的多模态 LLM 读取图片并返回答案。
- 答案标注表格所在页码和条款。

### 6.5 用户故事 5：生成测试清单

> 作为一名认证工程师，当我需要基于标准制定测试计划时，我希望 AI 能自动提取所有测试相关要求并生成结构化清单，我可以直接导出到 Excel。

**验收标准**：

- 一键生成测试清单。
- 清单字段完整：测试项、条款、条件、限值、判定、备注。
- 支持导出 Excel 和 Markdown。

## 7. 交互设计

### 7.1 主界面布局

采用 **双栏布局**，顶部为 PDF Tab 栏（支持多文件，最多 10 个），左右分栏宽度可鼠标拖拽调节，每个面板均可独立隐藏/显示：

```
┌──────────────────────────────────────────────────────────────┐
│  SpecReader AI    [Open PDF]                               │
├──────────────────────────────────────────────────────────────┤
│  [file1.pdf] [file2.pdf] [file3.pdf] ...                     │
├──────────────────────────────┬───────────────────────────────┤
│  [隐藏] [连续/单页]  [缩放]   │  [设置] [隐藏]                │
│                              │  [解读记录]                   │
│     PDF 阅读区（主区域）      │  ┌─────────────────────────┐  │
│     ┌──────────────────┐     │  │ 第 3 页  ▶              │  │
│     │ 选中段落...       │     │  │ 原文摘要...              │  │
│     │ [译] [解] 标记    │     │  └─────────────────────────┘  │
│     └──────────────────┘     │  │ 第 5 页  ▼              │  │
│                              │  │ 原文摘要...              │  │
│                              │  │ 解读内容 Markdown...     │  │
│                              │  └─────────────────────────┘  │
└──────────────────────────────┴───────────────────────────────┘
        ▲                              ▲
        └──── 拖拽调节宽度 ────────────┘
```

- 顶部 Tab 栏支持同时打开多个 PDF，最多 10 个；点击切换，点击 × 关闭。
- 默认左栏占 60%，右栏占 40%；最小宽度 240px。
- 两栏之间拖拽分隔条，实时调整宽度。
- 每个面板标题栏提供「隐藏」按钮；隐藏后收缩为边缘竖条，点击可恢复显示。
- 单栏显示时自动撑满剩余空间。
- 右侧面板“解读记录”按创建时间倒序展示解读条目，默认折叠，点击展开并跳转 PDF 对应位置。

### 7.2 选区浮动工具栏

用户在 PDF 中选中文本后，在选区上方出现浮动 Mini Toolbar：

```
[加入暂存] [解读] [翻译]
```

- 点击「加入暂存」将当前选中文本加入右侧面板暂存区，同时在 PDF 页面上生成橙色暂存标记。
- 点击「翻译」后在 PDF 页面上生成可拖动、可隐藏、可删除的绿色翻译浮层批注，流式显示翻译结果。
- 点击「解读」后在 PDF 页面上生成蓝色解读标记，右侧面板新增一条解读记录。
- 工具栏即时出现，不遮挡选中内容；点击页面其他任意位置工具栏自动消失。

### 7.3 答案引用设计

每个 AI 回答底部显示来源信息：

> 📄 来源：Clause 4.2.1 (p.23) · Clause 4.3 (p.25)

- 来源以 pill 形式展示。
- 点击来源跳转到 PDF 对应位置。
- 若来源为多个，列出 Top 3，更多可展开。

### 7.4 悬停取词翻译

鼠标悬停在英文单词上约 500ms 后，弹出迷你 tooltip：

```
┌─────────────────────────┐
│ electrostatic           │
│ /ɪˌlektrəʊˈstætɪk/      │
│ adj.                    │
│ • 静电的；静电学的       │
└─────────────────────────┘
```

- tooltip 跟随鼠标或显示在单词上方，移开鼠标立即消失。
- 使用本地 ECDICT 离线词典，不调用 LLM。
- 设置中可开启/关闭；首次开启若本地无词典，提示下载（支持断点续传）。

> 注：当前实现基于通用英汉词典，不等同于后续规划的「结构化术语表」（自动识别标准中的 Terms and definitions 并关联条款来源）。

### 7.5 测试清单界面

右侧面板“测试清单”标签页：

- 顶部：生成/重新生成按钮
- 中部：可编辑表格

## 9. 非功能需求

### 9.1 性能

- PDF 渲染：文档打开后 5 秒内可阅读。
- 文本解析与 Clause 索引：100MB 文本型 PDF 在 30 秒内完成。
- AI 首字符响应：普通网络环境下 < 3 秒。
- UI 交互：选区工具栏 0.3 秒内出现，页面滚动流畅。

### 9.2 兼容性

- 支持 Windows 10/11、macOS 12+（Intel + Apple Silicon）。
- 支持常见文本型 PDF，基于 PDF 1.4 及以上规范。
- 不保证扫描件/图片型 PDF 可用。

### 9.3 隐私与安全

- PDF 文件内容不上传云端，仅在本地解析和存储。
- 仅用户主动发起的 AI 请求会发送文本片段给用户配置的 LLM API。
- 表格截图仅在用户需要精确读取时发送给支持视觉的多模态 LLM。
- API Key 本地加密存储（系统 Keychain 或安全存储）。当前「超轻量版」暂未实现，仅保存在前端 `localStorage`。
- 首次启动明确告知数据处理方式，需用户同意。

### 9.4 可扩展性

- 架构预留多文档索引能力。
- 数据模型预留团队共享字段（如 workspace_id、shared_by）。
- License 系统预留个人版/团队版分层。

### 9.5 可维护性

- 前后端分离：Tauri IPC 作为边界。
- 统一 LLM 客户端接口，便于适配新供应商。
- Prompt 模板外置，支持后续 A/B 测试和优化。
- Tools 定义外置，便于新增查询能力。

## 12. 风险与假设

### 12.1 风险

| 风险                         | 影响           | 应对策略                                                          |
| ---------------------------- | -------------- | ----------------------------------------------------------------- |
| LLM API 输出不稳定，出现幻觉 | 影响答案可信度 | Prompt 中强调“只基于用户提供的选中文档内容回答”，不要编造条款编号 |
| 用户 API Key 泄露            | 影响安全信任   | 使用系统安全存储，不在日志中打印                                  |

### 12.2 假设

- 用户能够自行获取并配置 OpenAI 兼容 API Key。
- 超轻量版主要处理文本型 PDF。
- 目标语言以中文为主，其他语言为辅助翻译方向。
- 本阶段不做结构化文档解析，AI 仅基于用户选中的文本片段回答。
