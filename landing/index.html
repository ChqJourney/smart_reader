痛点

读标准，不该这么累

IEC / ISO / EN / GB / UL / ASTM / IEEE……每份标准都是数百页的专业文本，反复查词、翻页、回找，耗时又费神。

术语与语言障碍

英文原版标准里的专业术语和缩写反复打断阅读节奏，查完词典，上下文已经断了。

条款定位难

几百页里找一条要求本就不易，"see Clause 5.2.3" 每出现一次，还要手动翻页、定位、再找回原来的位置。

理解留不住

上周解读过的条款，这周重看又要从头再来；写报告时想引用结论，却想不起出自哪一页。

常见问题

上手前，你可能想问

需要自备模型 API 吗？费用高吗？

需要。填入任意 OpenAI 兼容 API 即可（DeepSeek / Kimi / Qwen / OpenAI 等），按量计费、无订阅费；日常查阅标准的消耗通常每份仅几毛到几块钱。也可填入企业内网模型地址，零外部调用。

我的 PDF 会被上传吗？

不会。文件只在本地读取和渲染；只有你主动选中的文本片段会发送给你自己配置的 API。配合内网部署的模型，可以做到数据完全不出内网。

和直接把 PDF 丢给网页版 AI 有什么区别？

整篇上传有泄密风险，且对话关掉就没了。SpecReader AI 只发送你选中的片段，翻译和解读以批注形式留在原文页码上，重开文件自动恢复，随时可跳回出处。

支持哪些系统？下载哪个文件？

目前发布 Windows 版本：在 Releases 页面下载安装包 SpecReader-AI_*_x64-setup.exe，或免安装的 SpecReader AI v*.exe；安装后启动时自动检查更新。macOS 可从源码执行 npm run tauri-build 自行构建。
